# Origami Robot Simulation

基于 Godot 4.6 的八面折纸机器人水动力仿真项目。

A water-dynamics simulation of an 8-faced origami-inspired robot, built with Godot 4.6.

---

## Overview | 项目简介

本项目通过 [ACDC4Robot](https://github.com/q-qo-o/ACDC4Robot) 工具链生成折纸机器人模型，导出为 SDF（Simulation Description Format）格式，并在 Godot 4.6 中构建物理仿真场景。

This project generates an origami robot model via the [ACDC4Robot](https://github.com/q-qo-o/ACDC4Robot) toolchain, exports it as an SDF (Simulation Description Format) file, and builds a physics simulation scene in Godot 4.6.

**核心特性 | Key Features:**
- 8 面闭合运动链 | Closed-loop kinematic chain with 8 faces
- 网格几何浮力计算 | Mesh-geometry buoyancy (water-line triangle clipping)
- 正弦波关节驱动 | Sinusoidal wave joint actuation
- SDF → Godot 自动导入 | SDF → Godot automated importer
- 视频录制导出 | Built-in video export via Movie Maker

        └── face{0-7}.stl             # Binary STL files (8 triangles each)

## Scene Architecture | 场景架构

### Main Scene (`scenes/origami_robot.tscn`)

| Component | Count | Description |
|-----------|-------|-------------|
| `RigidBody3D` | 8 | `face0`–`face7`, mass = 1.325 kg each |
| `HingeJoint3D` | 8 | Closed loop: `face0→…→face7→face0` |
| `MotionController` | 1 | Sinusoidal velocity profile for all joints |

- **Visual**: `MeshInstance3D` populated from binary STL via `STLParser`, each face has a unique color.
- **Collision**: `ConvexPolygonShape3D` via `ArrayMesh.create_convex_shape()`.
- **Buoyancy**: `buoyancy_mesh.gd` clips triangles against the water plane `y = 0` and applies Archimedes force.

### Outer Scene (`scenes/ocean.tscn`)

- **WaterSurface** — flat semi-transparent plane at `y = 0`
- **Ground** — `StaticBody3D` at `y = -0.05` for landing
- **DirectionalLight3D** — with shadows
- **Camera3D** — positioned at `y ≈ 1.0`, looking down

## Coordinate Conversion | 坐标转换

SDF → Godot:

| Axis | SDF | → | Godot |
|------|-----|---|-------|
| X | X | → | X |
| Y | Y | → | -Z |
| Z | Z | → | Y |

Basis: `Basis(Vector3(1,0,0), Vector3(0,0,-1), Vector3(0,1,0))`

## Video Export | 视频导出

```bash
# 1. Render to AVI (e.g., 12 seconds @ 60 FPS = 720 frames)
# DO NOT use --headless — Movie Maker requires real display driver.
GODOT="/c/Users/qdsyq/MyApps/Godot/Godot_v4.6.3-stable_mono_win64_console.exe"
mkdir -p output/videos
"$GODOT" --path . --write-movie output/videos/ocean_temp.avi --quit-after 720

# 2. Transcode to MP4
TIMESTAMP=$(date +%Y-%m-%d_%H-%M-%S)
ffmpeg -y -i output/videos/ocean_temp.avi \
  -c:v libx264 -preset medium -crf 18 \
  -pix_fmt yuv420p -movflags +faststart \
  "output/videos/ocean_${TIMESTAMP}.mp4"

# 3. Cleanup
rm output/videos/ocean_temp.avi
```

## Key Technical Details | 关键技术细节

- **Physics Engine**: Jolt Physics
- **Rendering**: Forward Plus, d3d12
- **Mesh Scale**: 0.001 (SDF default, millimeter-modeled meshes)
- **Material Cull Mode**: `CULL_DISABLED` — thin-shell origami faces are visible from both sides
- **Motor API**: `joint.set("motor/target_velocity", val)` — Godot 4.6 nested property path
- **Headless Mode**: HingeJoint3D motors are auto-disabled when `DisplayServer.get_name() == "headless"`
